\chapter{Introdução}
A proposta deste trabalho é implementar uma infraestrutura baseada em minitransações e avaliar o impacto da utilização desta infraestrutura no desenvolvimento de aplicações distribuídas. As minitransações são uma extensão do protocolo 2PC que permite acessar atomicamente ou modificar condicionalmente dados distribuídos em várias máquinas, permitindo assim o compartilhamento de estado de forma consistente entre as máquinas que compõem o sistema distribuído.

\subsection{Motivação}
Há diversos motivos para se construir uma aplicação que execute de forma distribuída em diferentes computadores --- tolerância a falha (adicionando mais máquinas ao sistema aumenta a chance de que, a qualquer momento, haja pelo menos uma máquina capaz de responder pelo sistema); escalabilidade (particionar e distribuir a carga de processamento do sistema permite que o sistema suporte um número crescente de usuários/informações); etc.

Uma vez que cada um dos computadores só tem uma visão parcial do que está acontecendo no sistema, é preciso que haja comunicação entre eles para que esse estado parcial seja expandido. Em geral essa comunicação ocorre através da troca de mensagens na rede entre os computadores para que os dados relevantes à execução - o estado do sistema - possa ser compartilhado e conhecido por todos. Esta abordagem é complexa e requer o uso de protocolos para gerenciar e manter consistente entre todas as máquinas o estado compartilhado entre o sistema.
